65  /* This is called whenever bytes_in_buffer has reached zero and more
66  data is wanted. In typical applications, it should read fresh data
67  into the buffer (ignoring the current state of next_input_byte and
68  bytes_in_buffer), reset the pointer & count to the start of the
69  buffer, and return TRUE indicating that the buffer has been reloaded.
70  It is not necessary to fill the buffer entirely, only to obtain at
71  least one more byte. bytes_in_buffer MUST be set to a positive value
72  if TRUE is returned. A FALSE return should only be used when I/O
73  suspension is desired (this mode is discussed in the next section). */
74  static boolean fill_input_buffer (j_decompress_ptr cinfo);
75 
76  /* Skip num_bytes worth of data. The buffer pointer and count should
77  be advanced over num_bytes input bytes, refilling the buffer as
78  needed. This is used to skip over a potentially large amount of
79  uninteresting data (such as an APPn marker). In some applications
80  it may be possible to optimize away the reading of the skipped data,
81  but it's not clear that being smart is worth much trouble; large
82  skips are uncommon. bytes_in_buffer may be zero on return.
83  A zero or negative skip count should be treated as a no-op. */
84  static void skip_input_data (j_decompress_ptr cinfo, long num_bytes);
